Глоссарий терминов

🔹 A

API (Application Programming Interface) – программный интерфейс приложения, позволяющий взаимодействовать различным программным компонентам.

AWS (Amazon Web Services) – набор облачных сервисов от Amazon, включающий вычисления, базы данных, хранение и инструменты машинного обучения.

Автоматизация документооборота – процесс внедрения технологий для управления документами без бумажного формата и ручного ввода данных.

🔹 B

Back-end – серверная часть системы, отвечающая за обработку данных и логику работы.

Big Data – технологии обработки и анализа больших объемов данных.

Blockchain – распределенная база данных, используемая для защиты информации от подделки.

🔹 C

CDN (Content Delivery Network) – сеть распределенных серверов для ускорения загрузки контента пользователями.

CI/CD (Continuous Integration / Continuous Deployment) – методы автоматизированной сборки, тестирования и развертывания кода.

Cloud Computing (Облачные вычисления) – использование удаленных серверов для хранения и обработки данных через интернет.

Containerization (Контейнеризация) – технология изоляции приложений с помощью контейнеров (Docker, Kubernetes).

🔹 D

DBMS (Database Management System, СУБД) – система управления базами данных (PostgreSQL, MySQL).

DDoS (Distributed Denial of Service) – атака на сервер с целью перегрузки системы.

DevOps – методология разработки ПО, объединяющая процессы разработки и эксплуатации.

Docker – платформа для контейнеризации приложений.

Документооборот – процесс создания, хранения, передачи и обработки документов в организации.

🔹 E

Encryption (Шифрование) – процесс преобразования данных в защищенный формат.

Elasticsearch – поисковая система для обработки больших объемов данных.

🔹 F

Firewall (Файервол) – система защиты сети от несанкционированного доступа.

Front-end – клиентская часть приложения, с которой взаимодействует пользователь.

🔹 G

Git – система контроля версий, используемая для отслеживания изменений в коде.

GDPR (General Data Protection Regulation) – закон ЕС о защите персональных данных.

Google Cloud Platform (GCP) – набор облачных сервисов от Google.

🔹 H

HTTP (HyperText Transfer Protocol) – протокол передачи данных в сети интернет.

HTTPS (HyperText Transfer Protocol Secure) – защищенный протокол передачи данных с использованием шифрования TLS.

🔹 I

IAM (Identity and Access Management) – управление учетными записями и доступами пользователей.

IDS/IPS (Intrusion Detection/Prevention System) – системы обнаружения и предотвращения вторжений в сеть.

Infrastructure as a Service (IaaS) – модель облачных вычислений, где инфраструктура предоставляется в виде услуги.

🔹 J

JavaScript – язык программирования, используемый для создания веб-приложений.

JSON (JavaScript Object Notation) – формат обмена данными между клиентом и сервером.

🔹 K

Kubernetes – платформа для оркестрации контейнеров, автоматизирующая их развертывание и управление.

Kafka (Apache Kafka) – распределенная система потоковой передачи данных.

🔹 L

Load Balancer (Балансировщик нагрузки) – система распределения запросов между серверами для обеспечения отказоустойчивости.

Linux – семейство операционных систем с открытым исходным кодом, используемых в серверных решениях.

Logging (Логирование) – запись системных событий для мониторинга и диагностики.

🔹 M

Machine Learning (ML, Машинное обучение) – область искусственного интеллекта, обучающая алгоритмы анализировать данные и делать прогнозы.

Microservices (Микросервисная архитектура) – подход к разработке ПО, где система состоит из множества небольших сервисов.

MFA (Multi-Factor Authentication) – многофакторная аутентификация, требующая нескольких уровней подтверждения личности.

🔹 N

Network Security (Сетевая безопасность) – защита сетевых ресурсов от несанкционированного доступа.

Node.js – среда выполнения JavaScript на сервере.

NoSQL – база данных, отличающаяся от традиционных реляционных СУБД (MongoDB, Cassandra).

🔹 O

OAuth 2.0 – протокол аутентификации, используемый для безопасного входа через сторонние сервисы.

OpenID Connect – надстройка над OAuth 2.0, позволяющая аутентифицировать пользователей.

🔹 P

PaaS (Platform as a Service) – модель облачных вычислений, в которой разработчики получают платформу для развертывания приложений.

PostgreSQL – реляционная система управления базами данных.

Python – язык программирования, широко используемый в разработке веб-приложений и анализе данных.

🔹 Q

Query (Запрос) – команда, отправляемая в базу данных для получения или обработки информации.

🔹 R

RBAC (Role-Based Access Control) – модель управления доступом на основе ролей.

REST API – архитектурный стиль взаимодействия веб-сервисов, основанный на HTTP-запросах.

Redis – высокопроизводительное хранилище данных в оперативной памяти.

🔹 S

SaaS (Software as a Service) – модель облачных сервисов, где пользователи работают с приложением через интернет (Google Docs, Dropbox).

SQL (Structured Query Language) – язык запросов к реляционным базам данных.

SSL/TLS – технологии шифрования для защиты интернет-соединений.

🔹 T

Terraform – инструмент для автоматизированного развертывания облачной инфраструктуры.

Threat Modeling (Моделирование угроз) – процесс анализа возможных атак и уязвимостей системы.

Token (Токен) – временный ключ доступа к API или системе.

🔹 U

UML (Unified Modeling Language) – язык моделирования программного обеспечения.

User Experience (UX) – пользовательский опыт, связанный с удобством взаимодействия с системой.

User Interface (UI) – интерфейс пользователя, включающий дизайн и элементы взаимодействия.

🔹 V

Virtualization (Виртуализация) – технология создания виртуальных машин и серверов.

VPN (Virtual Private Network) – технология создания безопасного соединения через интернет.

🔹 W

Web Application Firewall (WAF) – фильтр, защищающий веб-приложения от атак.

WebSocket – протокол для установления двустороннего соединения между клиентом и сервером.

🔹 X

XSS (Cross-Site Scripting) – тип атаки на веб-приложения, связанный с внедрением вредоносных скриптов.

🔹 Y

YAML (Yet Another Markup Language) – формат хранения конфигурационных данных.

🔹 Z

Zero Trust – модель кибербезопасности, основанная на принципе «не доверяй никому по умолчанию».
